Building an Image Geolocation Engine

An actionable, step-by-step engineering guide to build a GeoSpy-style system that predicts where a photo was taken from its pixels alone — no EXIF, no GPS. Verified Hugging Face models, runnable code, and a phased plan with explicit "Definition of Done" gates.

0The Problem

Given one photo with metadata stripped, predict (latitude, longitude) of capture.

This is image geolocalization. The core idea (since Im2GPS, 2008) is statistical: train on millions of geotagged images so a model learns the mapping from visual appearance → location. GeoSpy/Graylark ("Raven") markets a "Large Geolocation Model" and combines four method families — retrieval, classification, hybrid, and large vision-language models — which is exactly the academic lineage we build on below.

1What the Model Reads

The cues geolocation models exploit. PIGEON's attention maps confirm the model learns these unsupervised.

🏛️ Architecture

Building styles, materials, roofing, urban density — region-specific.

🌿 Vegetation & soil

Flora, crops, soil color, geology → climate band & continent.

🛣️ Road infrastructure

Lane markings, bollards, guardrails, utility poles — highly diagnostic per country.

🔤 Text & signage

Language, script, fonts, business names. OCR + LVLMs read these directly.

🚗 Driving side / vehicles

Left/right traffic, plate shapes/colors, common car models.

☀️ Sun, sky & climate

Solar angle & shadows → latitude/time; haze, snow, sky tone → climate.

2The Four Paradigms

① Retrieval (VPR)

Match against a geotagged gallery

Embed query, k-NN in a DB of geotagged photos, transfer coords. Im2GPS → NetVLAD → DINOv2. + Pinpoint where coverage is dense. − Needs a huge DB; fails on unseen places.

② Classification (geocells)

Carve Earth into cells, classify

PlaNet (2016): split globe into multi-scale cells (Google S2), predict the cell. + Scalable, generalizes. − Accuracy capped by cell size.

③ Image↔GPS alignment

Continuous coords (GeoCLIP)

CLIP-style contrastive alignment of an image encoder + a GPS location encoder (random Fourier features). Earth as a continuous function. + Continuous coords, open-source, data-efficient.

④ Multimodal-LLM + RAG

Reason like an analyst (Img2Loc)

Retrieve similar geotagged images, feed candidate coords + image to a vision-LLM that reasons over cues and explains. + No training, interpretable, SOTA. Closest to modern commercial tools.

3SOTA Blueprint: PIGEON / PIGEOTTO

Stanford 2023 — the reference full pipeline (beat top-0.01% GeoGuessr players; weights withheld for safety). We reproduce its ideas in Phase 4.

Semantic geocells

GADM admin boundaries → OPTICS clustering → Voronoi tessellation. Balanced & meaningful cells.

Haversine-smoothing loss

Soften one-hot cell labels by great-circle distance to neighbors → learns geographic adjacency.

Multi-task contrastive pretrain

CLIP ViT-L/14-336 + synthetic captions (location, climate, compass, season, driving side).

Auxiliary heads

Köppen climate, month, temperature, precipitation, elevation, population → positive transfer.

Hierarchical refinement

Top-K geocells → refine to a point via image-embedding L2 distance to location clusters.

Biggest wins (ablation)

Panorama input & haversine smoothing dominate; refinement adds street-level hits.

7Evaluation (build this first)

Standard metric: % of predictions within distance radii + median great-circle error. Wire this up in Phase 0 so every later change is measurable.

import math

def haversine_km(a, b):                       # a,b = (lat, lon)
    R = 6371.0
    lat1, lon1, lat2, lon2 = map(math.radians, [a[0], a[1], b[0], b[1]])
    h = (math.sin((lat2-lat1)/2)**2
         + math.cos(lat1)*math.cos(lat2)*math.sin((lon2-lon1)/2)**2)
    return 2 * R * math.asin(math.sqrt(h))

THRESH = {"street":1, "city":25, "region":200, "country":750, "continent":2500}

def report(preds, gts):                        # lists of (lat,lon)
    d = sorted(haversine_km(p, g) for p, g in zip(preds, gts))
    median = d[len(d)//2]
    acc = {k: 100*sum(x <= v for x in d)/len(d) for k, v in THRESH.items()}
    return {"median_km": round(median,1), "acc_pct": acc}
# Reference: PIGEON ~40% @ 25km, ~44km median on Street View.

P1

MVP — pretrained models behind an API

✓ done
Goal: upload a photo → get ranked coordinates on a map. No training. ~days.
  1. GeoCLIP path (continuous coords):
from geoclip import GeoCLIP
model = GeoCLIP()
gps, probs = model.predict("query.jpg", top_k=5)   # → [(lat,lon)...], [conf...]
  1. StreetCLIP path (zero-shot region classification, fully HF-native):
from transformers import CLIPModel, CLIPProcessor
from PIL import Image
m = CLIPModel.from_pretrained("geolocal/StreetCLIP")
p = CLIPProcessor.from_pretrained("geolocal/StreetCLIP")
candidates = ["Paris, France", "Tokyo, Japan", "Cairo, Egypt", "..."]  # your label set
prompts = [f"A Street View photo in {c}." for c in candidates]
x = p(text=prompts, images=Image.open("query.jpg"), return_tensors="pt", padding=True)
probs = m(**x).logits_per_image.softmax(-1)[0]
guess = candidates[int(probs.argmax())]
  1. FastAPI endpoint — strip EXIF on ingest to prove "pixels only":
from fastapi import FastAPI, UploadFile
from PIL import Image
from geoclip import GeoCLIP
import io
app, model = FastAPI(), GeoCLIP()

@app.post("/locate")
async def locate(file: UploadFile):
    img = Image.open(io.BytesIO(await file.read())).convert("RGB")
    img.getexif().clear()                       # remove metadata
    img.save("/tmp/q.jpg")
    gps, probs = model.predict("/tmp/q.jpg", top_k=5)
    return [{"lat":float(a),"lon":float(b),"confidence":float(c)}
            for (a,b),c in zip(gps, probs)]
  1. UI: React + Leaflet/MapLibre — drop top-5 pins, show confidence.
DoD: POST /locate returns 5 coords for any image; map renders pins; metrics logged vs P0 baseline.
P2

Retrieval backbone — OSV-5M + FAISS

✓ engine built
Goal: own gallery for fine, street-level matching where coverage is dense.
  1. Get the corpus: osv5m/osv5m — train.csv has latitude,longitude (+ aux labels); images ship as zips under images/train.
  2. Encode gallery → embeddings (CLIP/DINOv2/StreetCLIP image encoder), L2-normalized:
import faiss, numpy as np, torch, open_clip, pandas as pd
from PIL import Image
model, _, pre = open_clip.create_model_and_transforms("ViT-L-14", pretrained="openai")
model.eval().cuda()
df = pd.read_csv("train.csv")                   # id, latitude, longitude, ...
coords, embs = [], []
for _, r in df.iterrows():
    im = pre(Image.open(f"images/train/{r.id}.jpg")).unsqueeze(0).cuda()
    with torch.no_grad():
        e = torch.nn.functional.normalize(model.encode_image(im), dim=-1)
    embs.append(e.cpu().numpy()); coords.append((r.latitude, r.longitude))
X = np.vstack(embs).astype("float32")
index = faiss.IndexFlatIP(X.shape[1])           # cosine via inner product
index.add(X); np.save("coords.npy", np.array(coords))
# At scale (>1M) switch to IndexIVFFlat / HNSW.
  1. Query = k-NN + similarity-weighted fusion of neighbor coordinates:
def locate(qemb, k=10):
    D, I = index.search(qemb.astype("float32"), k)   # qemb: (1, dim) normalized
    w = np.exp(D[0]); w /= w.sum()                    # softmax over cosine sims
    c = coords[I[0]]
    return (w[:, None] * c).sum(0)                    # weighted centroid (lat,lon)
  1. Optional re-rank top candidates with DINOv2 / image matching for street-level precision.
DoD: city- and street-tier accuracy beat P1 on the holdout; index persisted and queryable in <100ms.
P3

Reasoning & interpretability — Vision-LLM + RAG

planned
Goal (optional, subordinate): the retrieval engine/own model decides; the LLM only refines among its candidates and explains the cues — it can never override the gallery. Skip it entirely for a fully self-hosted engine.
  1. Retrieve top-K candidates (coords + thumbnails + any OSV-5M labels) from the FAISS gallery.
  2. OCR signage (EasyOCR / PaddleOCR) and pass extracted text as extra context.
  3. Prompt a vision-LLM (GPT-4V / Qwen2-VL / LLaVA) with the image + candidates:
prompt = f"""You are a geolocation analyst. The image plus these candidate
locations (from visually similar geotagged photos) are given:
{candidates}              # e.g. [{{"lat":..,"lon":..,"country":"..","city":".."}}]
OCR text found in image: {ocr_text}
Identify the single most likely location. Justify using concrete visual cues
(architecture, signage script/language, vegetation, road markings, driving side).
Return strict JSON: {{"lat": float, "lon": float, "confidence": 0-1, "reasoning": str}}"""
# send (image, prompt) to the vision-LLM, then json.loads(response)
  1. Aggregate: if the LLM coord is far from all candidates, fall back to the retrieval centroid; keep its reasoning text.
DoD: every prediction returns coords + a cue-based explanation; accuracy ≥ P2 on the holdout.
P4

Custom model — PIGEON-style multi-task

planned
Goal: an owned model that generalizes to unseen places. Needs A100-class GPUs & weeks.
  1. Semantic geocells: load GADM admin polygons (geopandas), merge adjacent cells to a min sample count, split dense cells with sklearn.cluster.OPTICS + scipy.spatial Voronoi. Target ~2,000 cells.
  2. Haversine-smoothing targets (soft labels over cells):
import torch
def soft_targets(cell_centroids, true_xy, tau=75.0):   # tensors of (lat,lon)
    d = haversine_torch(cell_centroids, true_xy)        # [num_cells], great-circle km
    return torch.exp(-(d - d.min()) / tau)              # PIGEON eq. 2
# loss = -(soft_targets * log_softmax(logits, dim=-1)).sum(-1).mean()
  1. Backbone + heads: openai/clip-vit-large-patch14-336, freeze it, add a linear head → N geocells, plus auxiliary heads using OSV-5M's columns: climate (Köppen, classification), drive_side, and regressions for elevation/population (join WorldClim/GHSL).
  2. Contrastive pretraining: build synthetic captions from metadata ("A photo in {region}, {country}. {climate} climate. People drive on the {drive_side}.") and continue-train CLIP.
  3. Refinement: per geocell, OPTICS-cluster the training points; at inference take top-K cells, pick the cluster minimizing image-embedding L2, then the best point in it.
  4. Train on OSV-5M (+ MP-16 / GLDv2 for diversity); evaluate on Im2GPS3k + GWS15k.
DoD: beats the P2 retrieval baseline on Im2GPS3k and the unseen-place GWS15k; ablate each component.
